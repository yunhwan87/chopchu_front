# Merge Notes - Location Card

## 대상
- `components/LocationManager.js`
- `screens/LocationScreen.js`
- `App.js`
- `components/LoginForm.js`

## 핵심 변경
- 상단 탭 재구성
  - `전체 / 요청중 / 확정 / 보류/취소`
  - `답변대기` 독립 탭 제거
- `요청중` 탭 필터 로직 확장
  - `요청중 + (코디/제작진/섭외지 답변대기)` 함께 표시
- `요청중` 탭 전용 필터 추가
  - `답변 대기 필터` 박스에서 상태별 카드만 조회 가능
  - 다른 탭에서는 필터 박스 미노출
- 장소 수정 상태 버튼 변경
  - `답변대기` 제거
  - `코디 답변대기중 / 제작진 답변 대기중 / 섭외지 답변 대기중` 추가
- 작성자 기본값 로직 변경
  - 요청사항 작성자 기본 `"요청"` 제거
  - 로그인 사용자명 기반 자동 입력
  - 댓글 작성자도 로그인 사용자명 자동 입력
- 로그인 사용자명 연동
  - 로그인 시 이메일 앞부분을 닉네임으로 사용
  - 헤더 우측 이름을 `(사용자명)`으로 표시
  - `App -> LocationScreen -> LocationManager`로 사용자명 전달

## 머지 시 확인 포인트
- 상태값 문자열 충돌 여부
  - `코디 답변대기중`
  - `제작진 답변 대기중`
  - `섭외지 답변 대기중`
- 탭 텍스트 충돌 여부
  - `보류/취소` 라벨 유지 확인
- 작성자 필드 기본값 충돌 여부
  - 기존 `"요청"` 기본값 복원되지 않도록 확인

## 수동 테스트
1. `Location` 탭 진입
2. 상단 탭이 `전체/요청중/확정/보류/취소`인지 확인
3. `요청중` 탭에서 `답변 대기 필터` 노출 확인
4. 필터에서 상태 선택 시 해당 카드만 보이는지 확인
5. 카드 수정 모달에서 상태 버튼 3종 확인
6. 요청/댓글 작성자 입력칸에 로그인 사용자명이 기본 노출되는지 확인
