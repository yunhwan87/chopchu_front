# 섭외관리 카드 작업 요약

## 목적
- 이전 채팅에서 진행하던 `섭외관리(LocationManager)` 카드 관련 작업과 현재 채팅에서 이어서 반영한 작업을 기능 중심으로 한 번에 확인하기 위한 문서입니다.

## 범위
- 핵심 대상: `components/LocationManager.js`
- 연동 대상: `App.js`, `screens/LocationScreen.js`, `components/LoginForm.js`

## 1) 이전 채팅에서 반영된 기능(요약)
- 카드 리스트 기능 확장
- 장소명/담당자 검색
- 상태 필터 탭(전체/요청중/확정/보류·취소 계열)
- 리스크만 보기 토글
- 카드 정보 확장(담당자, 날짜/시간, 비용, 선금 여부, 리스크)
- 동일 시간대 섭외 충돌 가능 표시
- 모달 입력 확장(담당자, 시간, 비용, 선금 여부, 리스크, 메모)
- 상태값 정규화(예: `"섭외 중"` -> `"요청중"`)
- 요청사항 스레드/댓글 구조 유지 및 확장

## 2) 현재 채팅에서 추가/수정된 기능

### A. 상태/필터 UX 조정
- 상단 탭에서 `답변대기` 독립 탭 제거
- 상단 탭 구성: `전체`, `요청중`, `확정`, `보류/취소`
- `보류/취소` 탭은 보류/취소 상태를 함께 노출
- `요청중` 탭에서 아래 상태를 함께 관리
  - `요청중`
  - `코디 답변대기중`
  - `제작진 답변 대기중`
  - `섭외지 답변 대기중`

### B. 요청중 탭 전용 “답변 대기 필터” 추가
- `요청중` 탭에서만 노출되는 필터 박스 추가
- 선택값
  - `전체`: 요청중 + 3개 답변대기 상태를 모두 표시
  - 개별 상태 선택 시: 해당 상태 카드만 표시
- 탭을 벗어나면 필터 상태 자동 초기화
- 추가 UX 조정
  - 현재 선택값이 `전체`일 때 드롭다운을 펼치면 전체 목록 대신 3개 대기 상태만 보여주도록 변경

### C. 장소 수정(모달) 상태 버튼 조정
- 기존 `답변대기` 단일 상태 제거
- 세분화 상태 추가
  - `코디 답변대기중`
  - `제작진 답변 대기중`
  - `섭외지 답변 대기중`

### D. 요청/댓글 작성자 기본값 로직 수정
- “요청사항 작성자” 기본 텍스트 `"요청"` 제거
- 로그인 사용자명이 있으면 작성자 칸 기본값으로 자동 주입
- 댓글 작성자 칸도 로그인 사용자명 자동 주입

### E. 로그인 사용자 연동(기초)
- 로그인 시 이메일 앞부분을 닉네임으로 추출해 앱 전역 사용자명으로 전달
- 상단 프로필 표시에 현재 로그인 사용자명 표시
  - 형태: `(사용자명)`
- `App -> LocationScreen -> LocationManager`로 `currentUserName` 전달

## 3) 현재 동작 기준(체크리스트)
- `요청중` 탭에서 일반 요청 + 답변대기 3종을 함께 볼 수 있다.
- `요청중` 탭의 답변 대기 필터에서 특정 상태만 골라 볼 수 있다.
- 장소 수정 모달 상태 버튼에 답변대기 3종이 보인다.
- 요청/댓글 작성자 칸에 로그인 사용자명이 기본으로 들어간다.
- 상단 프로필 표시가 `(로그인 사용자명)`으로 보인다.
